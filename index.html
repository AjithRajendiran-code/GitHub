Section 01

What is Git?

Git is a distributed version control system — software that keeps a complete history of every change you make to your code. Think of it like an unlimited "undo" for your entire project.

📸

Snapshots, not backups

Git takes a snapshot of your project at any moment you choose. You can return to any snapshot at any time.

🌿

Branches

Work on a new feature in a separate "branch" without touching the main code. Merge when it's ready.

🤝

Team-friendly

Multiple people can work on the same project simultaneously without overwriting each other's work.

🌐

Offline-first

Git works entirely on your local machine. Push to a server (like GitHub) only when you're ready.

💡

Real-life analogy: Git is like Google Docs "version history" — but for your entire coding project, way more powerful, and works even without internet!

Section 02

What is GitHub?

GitHub is a cloud platform that hosts your Git repositories online. It adds collaboration tools on top of Git — like pull requests, issues, and project boards.

☁️

Cloud Storage

Your code is backed up online. Access it from any device, anywhere in the world.

🔍

Code Review

Submit a "Pull Request" so teammates can review and discuss your changes before merging.

🐛

Issue Tracking

Report bugs and plan features using GitHub Issues — like a built-in to-do list for your project.

🚀

GitHub Pages

Host your static website for FREE directly from your repository. No server needed!

🔑

Key difference: Git is the tool you install on your computer. GitHub is the website where you store and share your Git repositories. Git ≠ GitHub!

Section 03

Install Git

1

Download Git

Visit git-scm.com/downloads and click the download for your OS (Windows / Mac / Linux).

2

Run the Installer (Windows)

Accept defaults. On the "Adjusting PATH" step, keep "Git from the command line and also from 3rd-party software" selected.

3

Verify Installation

Open Git Bash (Windows) or Terminal (Mac/Linux) and type:

Terminal / Git Bash
git --version
# Expected output: git version 2.x.x

# Configure your identity (do this once!)
git config --global user.name "Your Name"
git config --global user.email "you@email.com"

# Verify config
git config --list
⚙️

Use the same email you'll use for GitHub. This links your commits to your GitHub profile.

Create a GitHub Account

1

2

Section 05

Essential Git Commands

These are the commands you'll use every day as a developer. Learn these 10 and you're ready for real projects.

Command What it does Example Phase
git init Initializes a new Git repository in the current folder. Creates a hidden .git folder. git init Setup
git clone <url> Downloads an existing repository from GitHub to your local machine. git clone https://github.com/user/repo.git Setup
git status Shows which files have been changed, added, or deleted. Your best friend! git status Check
git add <file> Stages a file — puts it in the "ready to save" zone. Use git add . to stage all files. git add index.html
git add .		 Save
git commit -m Takes a snapshot of all staged files. The message describes what you changed. git commit -m "Add navbar" Save
git log Shows the history of all commits. Press q to quit. git log --oneline Check
git remote add origin Links your local repository to a GitHub repository. git remote add origin https://github.com/user/repo.git Remote
git push Sends your commits from local to GitHub. -u origin main sets default upstream. git push -u origin main Remote
git pull Downloads the latest changes from GitHub into your local machine. git pull origin main Collab
git branch Lists all branches. git branch feature creates a new branch. git branch
git branch feature-nav		 Collab
git checkout Switches to a different branch. Use -b to create and switch. git checkout -b feature-nav Collab
git merge <branch> Merges another branch into the current one. git merge feature-nav Collab
Section 06

Create Repo in GitHub UI and Push Files

Use GitHub website to create the repository first, then connect your existing project and push your files. No mkdir or touch needed.

🌐

Step 1: Open GitHub and create a new repository

github.com -> + -> New repository
🏷️

Step 2: Enter repo name and visibility

Example: my-webpage (Public or Private)
📄

Step 3: Keep repo empty and create it

Do not add README, .gitignore, or license
🧭

Step 4: Open your existing project folder in terminal

cd /path/to/your-project
🔧

Step 5: Initialize Git only if needed

git init
➕

Step 6: Stage and commit your files

git add . && git commit -m "Initial commit"
🔗

Step 7: Link remote and push

git remote add origin https://github.com/yourname/my-webpage.git
🚀

Step 8: Push to GitHub

git push -u origin main
Terminal Steps After Creating Repo in GitHub UI
# Step 1: Open your existing project folder
cd /path/to/your-project

# Step 2: Initialize Git repository (skip if already initialized)
git init
# Output: Initialized empty Git repository in .../.git/

# Step 3: Check status (nothing yet)
git status
# Output: On branch main (or master), list of untracked files

# Step 4: Stage ALL files
git add .

# Step 5: Commit with a message
git commit -m "Initial commit: Add project files"
# Output: [main (root-commit) abc1234] Initial commit...

# Step 6: Link to your GitHub repository (created in UI)
git remote add origin https://github.com/yourname/my-webpage.git

# Step 7: Rename branch to main if needed
git branch -M main

# Step 8: Push to GitHub
git push -u origin main
# Done! Visit github.com/yourname/my-webpage to see your code
⚠️

If you get an error about "main vs master" branch, run: git branch -M main before pushing.

Section 07

Collaborate with Friends

🤝 Collaborator (Push)

Owner adds your friend as a collaborator in Settings → Collaborators on GitHub.

Friend's terminal
# Clone the repo
git clone https://github.com/owner/repo.git
cd repo

# Make changes, then push
git add .
git commit -m "Add footer section"
git push origin main

⬇️ Pull Latest Changes

Before you start working each day, always pull the latest changes from GitHub to avoid conflicts.

Your terminal
# Always pull before you start!
git pull origin main

# Now make your changes...
git add .
git commit -m "Update navbar styles"
git push origin main
🌿

Best practice for teams: Each person works on their own branch. When done, open a Pull Request on GitHub so teammates can review before merging to main.

Section 08

Using GUI — VS Code

VS Code has built-in Git support. You don't always have to use the terminal!

🔀

Source Control Panel

Click the branch icon in the left sidebar (or press Ctrl+Shift+G on Windows/Linux, Cmd+Shift+G on Mac). You'll see all changed files listed here.

➕

Stage Changes

Hover over a file in the Source Control panel and click the + button to stage it. Same as git add.

✅

Commit

Type your commit message in the message box at the top and press Ctrl+Enter on Windows/Linux or Cmd+Enter on Mac. Same as git commit -m.

☁️

Push & Pull

Click the three dots (···) menu → Push or Pull. Or use the sync icon at the bottom status bar.

💡

You can also use GitHub Desktop — a free app with a visual interface for Git: desktop.github.com

Section 10

How to Host on GitHub Pages

GitHub Pages lets you publish static websites directly from a repository. Follow these steps to make your project live.

1

Push your website files

Ensure your repository contains index.html at the root (or in a docs folder) and is already pushed to GitHub.

2

Open repository settings

In your GitHub repo, go to Settings → Pages.

3

Select source branch

Under Build and deployment, choose Deploy from a branch, then select main and /(root). Click Save.

4

Wait for deployment

After 1–3 minutes, GitHub shows your live URL like https://yourname.github.io/repo-name/.

Quick Deploy Commands
# From your project folder
git add .
git commit -m "Prepare site for GitHub Pages"
git push origin main

# Then enable Pages in GitHub UI: Settings -> Pages
⚠️

If your site loads without CSS or images, check your file paths. For project pages, prefer relative paths like ./styles.css instead of absolute paths starting with /.

Section 11

Git & GitHub Interview Questions with Best Answers

Use these as speaking templates in interviews. Keep answers short, clear, and practical with one example from your own projects.

1) What is the difference between Git and GitHub?

Best answer: Git is a distributed version control tool I run locally to track code history. GitHub is a cloud platform that hosts Git repositories and adds collaboration features like pull requests, issues, and CI workflows.

2) What is a repository?

Best answer: A repository (repo) is a project folder that contains source code, commit history, and configuration. In Git, the hidden .git directory stores all tracked snapshots and metadata.

3) Explain git add, git commit, and git push.

Best answer: git add stages selected changes, git commit creates a local snapshot with a message, and git push publishes those commits to a remote repository like GitHub.

4) What is the difference between git fetch and git pull?

Best answer: git fetch only downloads remote updates and does not modify my working branch. git pull performs fetch and then merges or rebases into my current branch.

5) What is a branch and why do we use it?

Best answer: A branch is an independent line of development. I use feature branches to isolate work, reduce risk on main, and make code review easier through pull requests.

6) What is a merge conflict and how do you resolve it?

Best answer: A merge conflict happens when Git cannot auto-combine overlapping edits. I resolve it by reviewing conflict markers, choosing the correct final code, testing, then committing the resolution.

7) Merge vs Rebase?

Best answer: Merge preserves full branch history and creates a merge commit; it is safest for shared branches. Rebase rewrites commits onto a new base for cleaner linear history; I avoid rebasing public shared history.

8) What does git stash do?

Best answer: git stash temporarily shelves uncommitted changes so I can switch context quickly. Later I restore them with stash pop or stash apply.

9) What is .gitignore and why is it important?

Best answer: .gitignore tells Git which files or folders not to track, such as build output, secrets, and dependency folders. It keeps repositories clean, secure, and lighter.

10) What is a Pull Request on GitHub?

Best answer: A pull request is a proposal to merge changes from one branch to another. It supports review comments, checks, and approvals before code is merged.

11) How do you undo the last commit?

Best answer: If I want to keep changes staged, I use git reset --soft HEAD~1. To keep changes unstaged, I use git reset --mixed HEAD~1. If already pushed, I prefer git revert to avoid rewriting shared history.

12) What are GitHub Issues and GitHub Actions?

Best answer: GitHub Issues track tasks, bugs, and feature discussions. GitHub Actions automates workflows such as tests, linting, and deployment on events like push or pull request.

🎯

Interview strategy: For every answer, give one real example from your own repo. This shows practical understanding, not just memorized definitions.
